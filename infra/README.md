# DevTracker - Complete Infrastructure as Code

This directory contains the complete, production-ready infrastructure configuration for the DevTracker mega-project, built with 20+ years of DevOps best practices.

## 🎯 Overview

The infrastructure stack provides a complete, scalable, and secure deployment platform for DevTracker:

- **Infrastructure**: AWS VPC, EC2, Security Groups, VPC Flow Logs, IAM Roles
- **Container Orchestration**: Kubernetes (Kind) on single node with HA ready
- **Application**: DevTracker Backend (FastAPI) + Frontend (Vite)
- **Data Layer**: PostgreSQL (production-ready) + Redis (caching)
- **Ingress**: NGINX Ingress Controller with advanced routing
- **Monitoring**: Prometheus + Grafana + AlertManager
- **Logging**: Loki + Promtail for centralized log aggregation
- **CI/CD**: ArgoCD for GitOps-based deployments
- **Security**: Network policies, RBAC, Secrets management
- **High Availability**: Pod disruption budgets, auto-scaling, pod affinity

## 🚀 Quick Start

### Prerequisites

```bash
# Install required tools
brew install terraform ansible aws-cli kubectl helm

# Configure AWS credentials
aws configure

# Generate SSH key
ssh-keygen -t rsa -b 4096 -f ~/.ssh/SRE.pem
```

### Deployment (5 minutes)

```bash
# Clone and setup
cd DevTracker-mega-project

# Deploy infrastructure and applications
chmod +x deploy-devtracker.sh
./deploy-devtracker.sh init      # Initialize Terraform
./deploy-devtracker.sh apply     # Deploy everything

# Wait for completion (10-15 minutes)
# Then access the applications (see URLs below)
```

## 🛠️ Management Commands

### Terraform

```bash
cd infra/terraform

# Initialize backend
terraform init

# Plan deployment
terraform plan

# Apply deployment
terraform apply

# View outputs
terraform output

# Destroy infrastructure
terraform destroy
```

### Ansible

```bash
cd infra/Ansible

# Run all playbooks
ansible-playbook -i inventory/hosts.auto playbooks/main.yml -v

# Run specific phase
ansible-playbook -i inventory/hosts.auto playbooks/02-docker-setup.yml -v

# Check syntax
ansible-playbook -i inventory/hosts.auto playbooks/main.yml --syntax-check

# Dry run
ansible-playbook -i inventory/hosts.auto playbooks/main.yml --check
```

### Kubernetes

```bash
# View cluster status
kubectl cluster-info
kubectl get nodes -o wide

# View all resources
kubectl get pods -A
kubectl get svc -A
kubectl get ingress -A

# Check application status
kubectl get deployment,pod -n devtracker -o wide

# View logs
kubectl logs -f deployment/devtrackr-backend -n devtracker
kubectl logs -f deployment/devtrackr-frontend -n devtracker

# Port forwarding
kubectl port-forward -n monitoring svc/prometheus-operated 9090:9090
kubectl port-forward -n devtracker svc/backend-service 8000:8000

# Scale applications
kubectl scale deployment/devtrackr-backend --replicas=5 -n devtracker
```

The infrastructure is configured for automatic scaling:

- **Backend**: 2-10 replicas, scales at 60% CPU or 70% memory

Manual scaling:
```bash
kubectl scale deployment/devtrackr-backend --replicas=5 -n devtracker
```

## 💾 Backup & Recovery

### Automated Backups

```bash
# Run backup script
bash infra/scripts/backup.sh

# Backups are stored in:
# - /opt/devtracker/backup/ (on instance)
# - S3 bucket (auto-uploaded)
```

### Manual Database Backup

```bash
# PostgreSQL backup
kubectl exec -it postgres-0 -n database -- pg_dump -U devtracker -d devtrackr > backup.sql

# Redis backup
kubectl exec -it redis-0 -n cache -- redis-cli BGSAVE
```

### Disaster Recovery

```bash
# Restore from backup
kubectl apply -f backup/namespace.yaml
kubectl apply -f backup/database-backup.sql

# See backup.sh for detailed recovery procedures
```

## 🚨 Troubleshooting

### Common Issues

1. **Instance not accessible**
   ```bash
   # Check security group
   aws ec2 describe-security-groups
   
   # Check SSH key
   chmod 600 ~/.ssh/SRE.pem
   ```

2. **Kubernetes cluster issues**
   ```bash
   kubectl get events -A --sort-by='.lastTimestamp'
   kubectl describe node
   kubectl logs -f deployment/<name> -n <namespace>
   ```

3. **Pod not starting**
   ```bash
   kubectl describe pod <pod-name> -n <namespace>
   kubectl logs <pod-name> -n <namespace>
   ```

See [INFRASTRUCTURE_GUIDE.md](../INFRASTRUCTURE_GUIDE.md) for detailed troubleshooting.
